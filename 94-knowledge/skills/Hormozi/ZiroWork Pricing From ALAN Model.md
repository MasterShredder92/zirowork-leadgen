# ZiroWork Pricing From ALAN Model

## Pricing call

ZiroWork should not launch as cheap SaaS first. It should launch as a **done-with-you / managed enrollment admin system** with a setup fee, a monthly base, and optional performance upside once proof is clean.

ALAN’s model worked because it sold outcomes instead of tools. For ZiroWork, the outcome is not “automation.” The outcome is **more enrolled students with less front-desk labor and fewer dead leads**.

## Why ZiroWork should not copy ALAN exactly

ALAN could lean hard on “pay after they show up” because gyms and performance agencies were already used to appointment-show economics. Music schools are different. A trial lesson can show up and still not enroll. A student can enroll and cancel quickly. So ZiroWork’s value metric should move closer to actual school revenue.

The strongest ZiroWork value metric is not **lead**. It is not even only **trial booked**. It is:

> **Qualified enrollment work completed:** inquiry followed up, trial booked or revived, CRM updated, parent reminded, outcome logged, and next action handled.

For the early product, charge for the installed enrollment machine first. Add performance only when tracking is reliable.

## Recommended early pricing structure

| Package | Price | What it includes | Why this price makes sense |
|---|---:|---|---|
| Pilot Setup | **$1,500–$3,000 one time** | Train one workflow, connect inquiry source, connect scheduler/CRM path, build first tenant skill, run first proof loop. | Low enough for music schools to say yes, high enough to fund hands-on onboarding. |
| Monthly Ops Base | **$500–$1,500/month** | ZiroWork dashboard, workflow runs, lead follow-up/admin handling, logs, repair support, limited volume. | Replaces part-time admin waste and captures ongoing workflow value. |
| Performance Add-On | **$25–$75 per qualified trial booked/showed** or **$100–$300 per enrolled student** | Only after tracking is clean. Applies when ZiroWork touches the lead and creates measurable outcome. | Aligns price with value without selling raw leads. |
| Multi-location / Scale | **$2,000–$5,000/month + performance** | More locations, more workflows, more lead volume, more repair/support, reporting. | Larger schools have more leakage and more upside. |

## Best MVP pricing

The clean first offer should be:

> **ZiroWork Enrollment Front Desk Pilot**  
> **$2,500 setup + $750/month for 90 days**  
> We install one enrollment workflow, train it inside your real admin stack, follow up with new inquiries, book or route trials, update your CRM/sheet, and show you a weekly log of every lead touched.

This is better than pure performance pricing at the start because ZiroWork still needs real-world proof, workflow repair data, and tenant-specific training. Pure performance too early creates disputes before the tracking layer is mature.

## When to add performance pricing

Performance pricing should only be added after ZiroWork can prove these events:

| Event | Required proof |
|---|---|
| Lead received | Source, timestamp, consent/source note, tenant ownership. |
| ZiroWork touched lead | Message/action log, workflow run ID, operator/automation source. |
| Trial booked | Calendar/CRM proof, date, teacher/location, parent confirmation. |
| Trial showed | Staff or system confirmation. |
| Student enrolled | CRM/payment/school confirmation. |
| Attribution | ZiroWork touched the lead before the outcome. |

Until these events are tracked, performance pricing will create arguments. The product must log proof before charging for outcomes.

## What ALAN teaches ZiroWork

| ALAN lesson | ZiroWork translation |
|---|---|
| Do not sell a generic tool. | Sell an enrollment/admin outcome. |
| Avoid low-trust retainers. | Use setup + base + proof-backed performance. |
| Work the leads for the customer. | ZiroWork must execute the front-desk workflow, not just store tasks. |
| Charge after value is visible. | Add per-booked-trial or per-enrolled-student fees after tracking is reliable. |
| Show replacement economics. | Compare against missed enrollments and admin labor, not against CRM subscriptions. |

## Pricing justification math for music schools

A music school can usually justify ZiroWork if it helps capture even a small number of students that would otherwise leak.

| Assumption | Conservative value |
|---|---:|
| Monthly tuition per student | $150–$250 |
| Average retained months | 6–12 months |
| Gross revenue per new student | $900–$3,000 |
| One saved enrollment per month | Often enough to cover base pricing |
| Three saved enrollments per month | Makes ZiroWork obviously profitable |

The sales message should be simple:

> “If ZiroWork saves one student per month that your follow-up would have lost, it likely pays for itself. If it saves three, it becomes one of the highest-ROI tools in the business.”

## Final recommendation

Start ZiroWork as **sales-led, managed, and outcome-priced**. Do not launch a $99/month self-serve SaaS. That would attract weak customers, create support drag, and underprice the actual value.

The best first pricing path is:

1. **$2,500 setup** to install the tenant workflow.
2. **$750/month 90-day pilot** to run and prove the enrollment/admin loop.
3. After proof: **$1,500/month base** or **$750/month + performance fee**.
4. Later: build self-serve only after the workflow library and onboarding path are repeatable.
