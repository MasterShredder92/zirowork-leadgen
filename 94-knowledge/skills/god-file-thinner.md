# SKILL SPEC: God File Thinner (v1.1)

## 1. Trigger Conditions

**Manual:** User or orchestration layer explicitly executes the thinning command.

**Autonomous:** Invoked by the workflow harness when `audit_result.flags.god_files_found` is non-empty.

**Execution Constraint:** Always executes as a strict two-phase sequence:

| Phase | `dry_run` | Expected Input | Behavior |
|---|---|---|---|
| Phase 1 | `true` | `audit_result` | Analyze dependencies and return extraction blueprint. Writes are blocked. |
| Phase 2 | `false` | `audit_result` + `extraction_plan` | Execute the approved plan. Writes are live. |

The calling harness must never skip Phase 1. Phase 2 may only proceed if Phase 1 returns `thinner_status: "plan_ready"`.

## 3. Constraints & Behavioral Safeguards

**Two-Phase State Lock:** If `dry_run == false` and `extraction_plan` is absent, empty, or contains zero `resolvable: true` elements, abort immediately with `thinner_status: "failed"`.

**Zero Logic Change Rule:** Structural extraction only. Never alter function signatures, rename variables, change logic flow, add error handling, or create new abstractions.

**Shared Global Freeze:** Code blocks interacting with global scopes (`window.*`, `global.*`, `process.*`) must not be relocated. Mark `resolvable: false`.

**Circular Import Circuit Breaker:** If extracting a candidate would create a circular dependency, mark `resolvable: false`.

**Immutable Backups Rule:** Backup snapshots must use `[filename].[current_date].bak` suffix. Never overwrite an existing backup file.

**Token Boundaries:** All target paths must match verified locations in CLAUDE.md. If CLAUDE.md is absent or graded F, fail immediately.

**Deadlock Avoidance:** If Phase 1 produces zero `resolvable: true` candidates, set `thinner_status: "failed"`, set `requires_re_audit: false`, and remove `"god_file_thinner"` from `next_recommended_skills`.

**Schema Compliance:** During Phase 1 dry run, all output arrays (`files_created`, `backup_files`, etc.) must return `[]` rather than being omitted.

---

## 4. Execution Protocol

### Phase 1 — Dry Run

1. Load `audit_result.flags.god_files_found`. Confirm destination layout via CLAUDE.md.
2. Sniff import dialect (ESM vs CJS). Isolate extraction candidate boundaries.
3. Trace dependencies. Determine topological order. Mark circular/dynamic imports `resolvable: false`.
4. Assign `target_path` per CLAUDE.md layer map. Calculate `projected_shell_lines`.
5. Deadlock check: if zero resolvable candidates → `thinner_status: "failed"`, remove self from queue.
6. Return plan with all execution arrays as `[]`.

### Phase 2 — Live Execution

1. Validate `extraction_plan` structure. Verify all `target_path` values still exist on disk.
2. Write `[source_path].[current_date].bak` snapshots before first write.
3. Process candidates in reverse-dependency order: write extracted code → remove from god file → inject import using detected `module_resolution_dialect`.
4. Scan modified file for dangling references. Flag `thinner_status: "partially_thinned"` if found.
5. Update CLAUDE.md Folder Structure section with newly created files (only permitted router write).

---

## 5. Downstream Routing Logic

- `requires_re_audit: true` if `thinner_status` is `"partially_thinned"` or `"failed"`
- If god file still >300 lines after extraction → append `"god_file_thinner"` to `next_recommended_skills` (second pass)
- If `thinner_status == "thinned"` → `requires_re_audit: false`, allow `folder_governance_migration` to proceed
