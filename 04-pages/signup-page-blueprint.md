## Actual signup flow

| Order | Screen / State | User-facing question | Field IDs / state | Options / input type | Required / branch rule |
|---:|---|---|---|---|---|
| 0 | `step=0` | `Who is this for?` | `enrollmentType`, `forSelf` | `kid`, `kids`, `self`, `gift` | Required. `self` sets `forSelf=true`; `kid/kids` set `forSelf=false`; `gift` opens gift-recipient sub-screen. |
| 0A | `step=0`, gift branch | `Who's the lucky recipient?` | `giftRecipient` | Text input | Required only when `enrollmentType=gift`; then `forSelf=false`. |
| 1A | `step=1`, `step1Sub=0` | `Tell us about yourself.` or `Tell us about them.` | `studentFirstName`, `studentLastName`, `age` | First name, last name, numeric age 3-99 | First and last required. Age is collected but not blocking in the current UI. |
| 1B | `step=1`, `step1Sub=1` | `A bit more about you.` or `A bit more about {student}.` | `experience`, `freeText` | Experience pill + text area | Both required. |
| 2A | `step=2`, `step2Sub=0` | `What do you want to learn?` or `What does {student} want to learn?` | `selectedInstruments` | Multi-select instruments | At least one required. URL can preselect `piano`, `guitar`, `vocals`, or `drums`. |
| 2B | `step=2`, `step2Sub=1` | `What days work best?` | `selectedDays` | Multi-select day pills | At least one required. `Any of These Work` and `None of These Work` are exclusive choices. |
| 2C | `step=2`, `step2Sub=2` | `Do you have an instrument?` or `Does {student} have an instrument?` | `hasInstrument` | Single choice | Skipped when selected instruments are vocals-only. Otherwise selecting an option auto-advances. |
| 2D | `step=2`, `step2Sub=3` | `Your location` | `preferredLoc` | Auto-set from current site location | Not a typed answer. Shows the location based on the page the user came from. |
| 2E | `step=2`, `step2Sub=4` | `Any other locations work?` | `secondaryLocs` | Optional multi-select locations except preferred | Optional. User can hit `Next` or `Skip`. |
| 3 | `step=3` | `How do we reach you?` or gift contact question | `parentFirstName`, `parentLastName`, `email`, `phone`, `isMilitary` | Parent/gift giver name, email, phone, military checkbox | Parent/gift giver first and last required only when `forSelf=false`. Email and phone required for everyone. Email must pass format validation. |
| 4 | `step=4` | Additional student question | `additionalStudents` | `Just me/Just {student}` or `Add Another Student` | Optional. Allows adding more students until `additionalStudents.length < 3`, meaning primary + up to three additional students. |
| 4A | `step=4`, adding student | `Additional student #{n}` | `addStudentFirstName`, `addStudentLastName`, `addStudentInstruments`, `addStudentBio`, `addStudentGoals` | Text inputs, instrument multi-select, textareas | First name, last name, bio, and goals required. Instrument selection is captured but not blocking in current UI. |
| 5 | `step=5` | `One last thing — how did you hear about us?` | `referralSource` | Single choice | Required by flow because selecting an option advances to matching. |
| 6 | `step=6` | Match/loading and final submit | `matchScore`, `matchedTeacher` | Generated by public teacher match endpoint | User clicks `Complete Enrollment →`. |

## ZiroWork implication

The runtime JSON should not guess Square dropdown words like `Trial Lesson`. It should be generated from this intake schema first, then mapped into Square. For the MVP replay test, use a `kid` or `self` path with one student and avoid optional additional students until the base path fills cleanly.
