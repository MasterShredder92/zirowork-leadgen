## COMPONENT CONVENTIONS (per-file; non-negotiable)
1. File top: `"use client"` (all operator views are interactive)
2. CSS variables, not hex literals: `T.bg` → `var(--color-bg)`, `T.t1` → `var(--color-text-1)`, etc. (map below)
3. Import hooks from `@/hooks/tables` (not `window.*`)
4. Import icons from `lucide-react` (not `window.LucideReact`)
5. `window.sb` → `import { supabase } from '@/lib/supabase/client'`
6. `onMouseEnter` row-hover → Tailwind `hover:bg-[var(--color-row-hover)]` on `<tr>` (no inline style mutation)
7. tsc strict: no `any`, no untyped params
8. Gate structural check: `grep -rnE '\bwindow\.' src/components/` must return 0 hits

## RENDER-DIFF GATE (the invariant)
For every registered view, `verify-phase-3-views.sh` runs `render-diff.mjs` which:
1. Takes a screenshot of the Next.js view at `http://localhost:3000/<slug>`
2. Compares it to the committed baseline PNG at `_migration/epic/GATES/snapshots/<view>.png`
3. Exits non-zero if pixel diff > 1%

Baseline PNGs are generated ONCE from the legacy app (via `node render-diff.mjs baseline <view>`) and committed. They are never regenerated by the gate itself.

Gate goes RED if:
- The baseline PNG exists but the Next.js view screenshot differs by >1%
- The baseline PNG exists but the Next.js route returns non-200
- No ported views are found in the VIEWS array (initial state)

Gate goes GREEN if all registered views match their baselines within threshold.

**RED-TEST**: The gate is committed with `insights` in the VIEWS array BEFORE InsightsView is ported to Next.js. The gate must exit 1 at that point. After porting InsightsView, it exits 0. This is the proof-of-gate-correctness step.

## PER-VIEW DONE CRITERIA
A view is done when ALL of:
1. `npx tsc --noEmit` exits 0 (no type errors introduced)
2. `npx eslint .` exits 0 (no lint errors)
3. `npm run build` exits 0 (build succeeds)
4. Route `http://localhost:3000/<slug>` returns 200
5. `render-diff.mjs compare <view>` exits 0 (screenshot matches baseline within 1%)
6. No `window.*` in `src/components/<domain>/`

All 6 must be green. "Looks right" is not evidence.

## DEVIATIONS PROTOCOL
If any view cannot be ported identically (e.g., a behavior that requires a server component, or a third-party dep not available in ESM):
1. STOP and surface to Zach — do NOT approximate.
2. Describe the specific incompatibility.
3. Wait for a spec decision before continuing.
