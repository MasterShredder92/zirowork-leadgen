# CHECKPOINTS

One file per completed stage, named `<stage>.md`, written by the worker BEFORE declaring done,
using the schema in `../RUBRIC.md`. These are the adversarial AUDIT trail — what happened, what
was NOT verified, and a gaming disclosure that must read "none". They are not proof of
correctness (the gate is); they're how Zach reviews a stage in one screen instead of file-by-file.

Concatenated end-to-end at the finish, this folder is the forensic record of the whole rebuild.
